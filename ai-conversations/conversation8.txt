TECHNICAL DEEP DIVE: How Ollama Runs in Colab
-------------------------------------------------------------------------------

### The Challenge:

Google Colab notebooks run in ephemeral Linux VMs that:
- Reset when session ends (no persistence)
- Have restricted system access (can't install system services normally)
- Run as non-root user (limited privileges)

### The Solution:

**1. Installation Without Root:**
```bash
curl -fsSL https://ollama.ai/install.sh | sh
```
- Ollama's install script handles non-root installation
- Installs to user's home directory
- No sudo required

**2. Background Service:**
```python
subprocess.Popen(
    ["ollama", "serve"],
    stdout=subprocess.DEVNULL,
    stderr=subprocess.DEVNULL
)
```
- Starts Ollama as background process
- Doesn't block notebook execution
- Runs for duration of Colab session

**3. Model Storage:**
```bash
ollama pull phi  # Downloads to ~/.ollama/models/
```
- Models stored in user directory
- Persists within session
- Must re-download if session expires

**4. Communication:**
```python
import ollama
response = ollama.chat(model="phi", messages=[...])
```
- Python client connects to local Ollama service
- Uses localhost:11434 (default Ollama port)
- Works exactly like local installation

### Performance Considerations:

**Colab Free Tier:**
- CPU: ~2 cores
- RAM: ~12GB
- GPU: Optional (T4 when available)

**Model Performance:**
- **phi**: ~2-3 seconds per response (CPU)
- **llama3.2**: ~3-5 seconds per response (CPU)
- With GPU: 2-3x faster

**Comparison:**
```
Keyword-based: <0.1 seconds (instant)
Ollama (phi):   2-3 seconds (thoughtful)
Ollama (llama): 3-5 seconds (more capable)
```

### When to Use Each Approach:

**Use Ollama in Colab when:**
- ✅ Demonstrating full project capabilities
- ✅ Testing complex natural language queries
- ✅ Showing AI-powered features to others
- ✅ Time is not critical (2-5 min setup acceptable)

**Use Keyword Fallback when:**
- ✅ Quick testing of other features
- ✅ Teaching basic concepts
- ✅ Limited Colab resources
- ✅ Simple queries that don't need AI
