A. Synchronous Blocking Operations
   - Problem: main() loop blocks on every operation
   - Impact: Poor user experience, no concurrent operations
   - Solution: Implement async/await pattern

B. Fragile String Parsing
   Location: lines 28-30 in run_simple_agent()
   ```python

   end = content.find("'", start)
   ```
   - Problem: Brittle LLM output parsing
   - Impact: Will break if format changes
   - Solution: Use structured output (JSON mode) or regex

C. Inefficient Data Fetching
   - Problem: Weather data fetched on every menu cycle
   - Impact: Unnecessary API calls, slow response
   - Solution: Cache weather data object in memory

D. Limited Error Handling
   - Problem: Minimal try-catch, no retry logic
   - Impact: App crashes on transient failures
   - Solution: Implement retry with exponential backoff

E. Agent Iteration Limit
   - Problem: max_iterations=5 may be insufficient
   - Impact: Complex queries may fail prematurely
   - Solution: Adaptive iteration strategy

F. Chart Generation
   - Problem: Matplotlib plt.show() is blocking
   - Impact: Freezes UI during chart display
   - Solution: Use non-blocking backend or save to file

3. OLLAMA CONNECTION ISSUE - DIAGNOSIS
--------------------------------------

ERROR: "Connection error." when using AI features (option 4)

ROOT CAUSE:
- Container Environment Detected:
  * Hostname: 793a1217a4a9
  * Localhost: 127.0.0.1
  * Environment: Google Colab or Docker container

- Problem Explanation:
  The code connects to: http://localhost:11434/v1
  In a container, "localhost" refers to the container itself, not the host machine.
  Ollama is running on the HOST machine, not inside the container.
  Therefore, the container cannot reach Ollama via localhost.

SOLUTIONS:

Option A: Docker Desktop (if using local Docker)
----------------------------------------
Replace:
  base_url="http://localhost:11434/v1"
With:
  base_url="http://host.docker.internal:11434/v1"

Option B: Google Colab Environment
-----------------------------------
Colab cannot access local machine services. Options:
1. Use cloud-hosted LLM (OpenAI, Anthropic, etc.)
2. Use ngrok to tunnel Ollama:
   - Install ngrok on host
   - Run: ngrok http 11434
   - Use ngrok URL in base_url
3. Run notebook locally instead of in Colab

Option C: Linux Docker
----------------------
Replace:
  base_url="http://localhost:11434/v1"
With:
  base_url="http://172.17.0.1:11434/v1"  # Docker bridge network

Option D: Verify Ollama is Running
-----------------------------------
On host machine terminal:
  curl http://localhost:11434/api/tags

Should return list of available models.
